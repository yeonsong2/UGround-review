# 논문 리뷰: UGround

> **Navigating the Digital World as Humans Do: Universal Visual Grounding for GUI Agents**
> Boyu Gou et al. | ICLR 2025 Oral (상위 1.8%) | [arXiv](https://arxiv.org/abs/2410.05243) | [GitHub](https://github.com/OSU-NLP-Group/UGround) | [Demo](https://huggingface.co/spaces/orby-osu/UGround)

---

## 1. 한 줄 요약

> AI가 화면을 **눈으로만 보고** UI 요소의 정확한 위치(픽셀 좌표)를 찾아내는 범용 모델

---

## 2. 문제 정의

### 기존 방법의 한계

GUI 에이전트(화면을 제어하는 AI)는 보통 두 가지 방식으로 동작했다.

| 방식 | 설명 | 한계 |
|-----|-----|-----|
| **텍스트 기반** | HTML 코드, 접근성 트리(Accessibility Tree) 분석 | 모든 앱에서 사용 불가, 구조가 없는 화면에서 실패 |
| **시각 기반 (기존)** | 화면 이미지를 직접 분석 | 학습 데이터 부족 → 성능 낮음 |

### 이 논문이 해결하려는 것

```
입력: 화면 스크린샷 + 자연어 설명 ("검색 버튼을 클릭해줘")
출력: 픽셀 좌표 (x, y)
```

사람처럼 **화면만 보고** UI 요소를 찾는 모델을 만든다.

---

## 3. 핵심 기여 (Contribution)

### 3-1. UGround 모델

- **Qwen2-VL** 아키텍처 기반의 멀티모달 모델
- 3가지 크기: 2B / 7B / **72B**
- 출력: 0~1000 범위의 정규화 좌표 → 실제 픽셀로 변환

```
모델 출력 예시: (578, 421)
실제 좌표 변환: x = 578/1000 × 이미지 가로, y = 421/1000 × 이미지 세로
```

### 3-2. 대규모 합성 데이터셋

| 항목 | 수치 |
|-----|-----|
| 스크린샷 수 | **130만 장** |
| GUI 요소 수 | **1,000만 개** |
| 주요 출처 | 웹 페이지 (95% 이상) |
| 특징 | 웹 데이터만으로 모바일/데스크톱까지 일반화 |

### 3-3. SeeAct-V 프레임워크

UGround를 실제 GUI 에이전트에 연결한 시스템.
- LLM이 "무엇을 할지" 결정 → UGround가 "어디를 클릭할지" 결정
- 텍스트 구조(HTML 등) 없이 순수 시각 정보만 사용

## 4. 모델 구조

```
┌─────────────────────────────────────────────────┐
│                   입력                           │
│   화면 이미지  +  "검색 버튼을 클릭해줘"           │
└────────────────┬────────────────────────────────┘
                 │
         ┌───────▼────────┐
         │  Vision Encoder │  ← 이미지를 토큰으로 변환
         │  (Qwen2-VL)    │
         └───────┬────────┘
                 │
         ┌───────▼────────┐
         │  LLM Decoder   │  ← 좌표 예측
         │  (Qwen2)       │
         └───────┬────────┘
                 │
┌────────────────▼────────────────────────────────┐
│                   출력                           │
│   (578, 421)  →  실제 픽셀 좌표로 변환            │
└─────────────────────────────────────────────────┘
```

## 5. 실험 결과

### 5-1. ScreenSpot 벤치마크 (주요 결과)

ScreenSpot은 GUI grounding의 표준 벤치마크 (모바일/데스크톱/웹 × 텍스트/아이콘).

| 모델 | 평균 정확도 |
|-----|-----------|
| GPT-4o | 18.3% |
| Claude-3.5 Sonnet | 21.1% |
| SeeClick (기존 SOTA) | 47.4% |
| **UGround-V1-7B** | **76.3%** |
| **UGround-V1-72B** | **89.4%** |

→ 기존 최고 모델 대비 **최대 42%p 향상**

### 5-2. 우리가 직접 재현한 결과 (UGround-V1-7B, 300샘플)

| 카테고리 | 정확도 |
|---------|-------|
| mobile_text | 100.0% |
| mobile_icon | 84.0% |
| desktop_text | 92.0% |
| desktop_icon | 84.0% |
| web_text | 94.0% |
| web_icon | 78.0% |
| **평균** | **88.67%** |

✅ 논문 보고값(89.4%)과 거의 동일하게 재현 성공

### 5-3. 데모: 직접 만든 UI 화면에서 테스트

| 자연어 질의 | 예측 좌표 | 결과 |
|-----------|---------|-----|
| "the search button" | (739, 30) | ✅ 정확 |
| "the login button" | (1204, 29) | ✅ 정확 |
| "the like button" | (110, 385) | ✅ 정확 |
| "the search input field" | (500, 30) | ✅ 정확 |

## 6. 한계점

| 한계 | 설명 |
|-----|-----|
| **학습 데이터 편향** | 웹 데이터 중심 → 특수한 UI에서 성능 저하 가능 |
| **동적 화면** | 애니메이션, 팝업 등 시간에 따라 변하는 UI에 취약 |
| **소형 요소** | 매우 작은 아이콘은 정확도 낮음 (web_icon 78%) |
| **언어 의존성** | 영어 기반 학습 → 다국어 UI에서 성능 미검증 |

---

## 7. 의의 및 결론

1. **범용성**: 모바일, 데스크톱, 웹 모두에서 하나의 모델로 동작
2. **데이터 효율**: 웹 합성 데이터만으로 다른 플랫폼까지 일반화
3. **실용성**: 실제 GUI 에이전트(SeeAct-V)에 바로 적용 가능
4. **재현성**: 공개 모델 + 데이터셋 → 누구나 재현 가능

> GUI 에이전트 분야에서 "시각 기반 접근"의 가능성을 강하게 입증한 논문.
> 향후 RPA(로봇 프로세스 자동화), 접근성 도구, AI 에이전트 시스템에 직접 활용 가능.

## 8. 환경 및 재현 방법

```bash
# 환경 설치
conda create -n uground python=3.10 -y
conda activate uground
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
pip install transformers accelerate qwen-vl-utils pillow huggingface_hub datasets

# 모델 다운로드
python -c "
from huggingface_hub import snapshot_download
snapshot_download('osunlp/UGround-V1-7B', local_dir='models/UGround-V1-7B')
"

# 데모 실행
python demo/run_demo_발표용.py

# ScreenSpot 평가
python demo/run_screenspot_발표용.py
```
